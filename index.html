Learn Docker Fundamentals

What is Docker?

Docker is a platform for developing, shipping, and running applications in containers. Containers package up code and all its dependencies so the application runs quickly and reliably from one computing environment to another.

Key Concepts:

  • Container: A lightweight, standalone executable package
  • Image: A read-only template used to create containers
  • Dockerfile: A text file with instructions to build an image
  • Registry: A repository for storing Docker images

Docker Images

Images are the building blocks of Docker. They contain the application code, libraries, and dependencies needed to run your application.

# Pull an image from Docker Hub
docker pull nginx:latest

# List all images
docker images

# Remove an image
docker rmi <image-id>

Docker Containers

Containers are running instances of Docker images. They are isolated environments that run your applications.

# Run a container
docker run -d -p 80:80 nginx

# List running containers
docker ps

# Stop a container
docker stop <container-id>

# Remove a container
docker rm <container-id>

Docker Networking

Docker networking allows containers to communicate with each other and the outside world.

# Create a network
docker network create my-network

# List networks
docker network ls

# Run container on specific network
docker run --network my-network nginx

Docker Volumes

Volumes persist data generated by and used by Docker containers, independent of the container lifecycle.

# Create a volume
docker volume create my-volume

# List volumes
docker volume ls

# Mount volume to container
docker run -v my-volume:/data nginx

Dockerfile

A Dockerfile is a text document that contains all the commands a user could call on the command line to assemble an image.

FROM node:16-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]

Command Reference

Container Management

docker run Create and run a container
docker ps List running containers
docker stop Stop a running container
docker start Start a stopped container
docker rm Remove a container
docker exec Execute command in running container
docker logs View container logs

Image Management

docker images List all images
docker pull Download an image
docker build Build image from Dockerfile
docker push Upload image to registry
docker rmi Remove an image
docker tag Tag an image

Network & Storage

docker network create Create a network
docker network ls List networks
docker volume create Create a volume
docker volume ls List volumes
docker inspect Get detailed information

Common Flags

-d Detached mode (background)
-p Port mapping
-v Volume mount
--name Container name
--network Connect to network
-it Interactive terminal
--rm Auto-remove after exit
